DEVELOPMENT PATH

从需求到结果的完整开发路径

按研发执行顺序记录从数据整理到网页端联调的关键节点，便于理解系统落地路径。

  1. 01

    数据集准备

    按叶片与果实两类场景分别整理训练数据，叶片方向覆盖复杂病虫害纹理，果实方向聚焦典型果面异常特征。

  2. 02

    YOLO格式标注

    结合类别完整性、图像质量与场景多样性检查，建立 YOLO 标签文件与检测框坐标。

  3. 03

    训练配置

    通过 data.yaml 配置训练集、验证集、测试集路径以及所有类别名称。

  4. 04

    模型训练

    使用 Ultralytics YOLO26 目标检测流程进行训练，让模型同时学习病害类别、纹理特征与目标位置。

  5. 05

    验证与导出

    训练结果保存到 runs/detect，生成 best.pt 和 last.pt，并选用 best.pt 导出 ONNX。

  6. 06

    网页端联调

    后端加载 ONNX 模型，接入两种图像增强检测模式，提升识别稳定性，并完成结果封装与前端展示。

SYSTEM LOGIC

系统如何完成一次检测

一次检测由上传、校验、接口接收、模型推理、结果封装和前端反馈组成。

01

输入准备

用户提交叶片或果实图像，前端完成预览、格式校验与检测准备。

  • 用户上传图片
  • 前端预览与校验
  1. 用户上传图片 用户选择待检测图片，页面进入预览与检测准备状态。
  2. 前端预览与校验 前端读取图片信息，完成格式校验、预览渲染和检测按钮状态更新。
02

识别处理

后端接收图片并调用 ONNX Runtime 推理，再输出类别、检测框与结构化结果。

  • /detect 接收图片
  • ONNX Runtime 推理
  • 输出类别与检测框
  • 结果结构化封装
  1. /detect 接收图片 Flask 接口接收上传图片、检测类型与增强模式，并进入统一检测流程。
  2. ONNX Runtime 推理 后端调用已接入的 ONNX 模型，对图像中的疑似病害区域进行推理。
  3. 输出类别与检测框 模型输出被整理为类别名称、置信度和检测框坐标，供前端绘制标注。
  4. 结果结构化封装 后端将检测结果转换为统一 JSON，包含摘要、建议、框选信息和历史记录字段。
03

结果反馈

前端展示识别结果，并沉淀历史记录与建议反馈，方便回看和复盘。

  • 前端展示识别结果
  • 历史记录与建议反馈
  1. 前端展示识别结果 页面根据返回结果绘制识别框，更新结论、建议、置信度和检测状态。
  2. 历史记录与建议反馈 检测完成后将结果加入历史记录，并保留处理建议，方便样例对比与过程复盘。

可扩展

后续可继续扩展类别、替换更高精度权重或训练更多作物模型。

可维护

前后端接口结构清晰，检测流程可运行，便于继续开发和问题定位。

可复盘
